# ①課題名
ローカルストレージを活用した不可逆的くじ引き

## ②課題内容（どんな作品か）
- とにかく沢山くじを引きたいと思っても、前回まではブラウザに全結果を記載したため、レスポンスの低下につながっていました。
- そこで、ボタン⓶を新たに実装しました。ローカルストレージに一定回数の結果を保存することで、ブラウザへの負荷を最低限に沢山くじを引けるようにしました。結果の配列は、slice(-10)を活用し、10のみ常に保持し、古いデータは削除されて常に新しいデータに置き換わるようにしています。
- また、保有資産が減っていく感じが見えないと面白くないと思ったので、ローカルストレージ最初に資産額を入れてゲームを始めるとデータが保存されて保有資産の金額をいじれないままくじを引き続ける他なくなるボタン⓷を実装しました。借金が５回生じるとゲームオーバー（ローカルストレージを削除しない限り未来永劫ボタンをクリックできなくなる）になるギミックも入れました！

## ⑤工夫した点・こだわった点
- ローカルストレージは3か所で使用しています。
- １つ目）ボタン⓶の10個のデータのみを保持してグラフにする配列保存用途。
- ２つ目）1度でもボタン⓷を一度でもクリックすると保有資産額を操作できなくなるギミックのフラグ立てとデータ保存用途。
- ３つ目）ボタン⓷で借金をした回数を保存してゲームオーバー判定をする用途。
- ３等賞以上は、AIで作った特別な動画が流れるようにifを使っています。AIサービスの操作が難しく、これで2日の休みを消費してしまいました。

## ⑥難しかった点・次回トライしたいこと（又は機能）
- ローカルストレージ内のデータを使い、ボタンや入力エリアの制限を付与するギミックに苦労しました。
- データベースを活用することで出来ることがもっと増えそうなので、保存した大量のデータを利用したギミックなどを作っていきたいです。
